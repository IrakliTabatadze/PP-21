git init - აკეთებს გიტის ინიციალიზაციას, პროექტში შექმნის .git საქაღალდეს, რომელიც არის ლოკალური რეპოზიტორი

git add --all/. - დაამატებს ყველა ფაილს მონიტორინგისთვისი რეპოზიტორიში, შეგვიძლია მივუთითოთ პირდაპირ ფაილის სახელი

git commit -m '' - აკომიტებს ცვლილებებს, ვამატებთ კომენტარს თუ რა ცვლილებები შევიტანეთ

git log - გვაჩვენებს კომიტების ისტორიას

git status - გვაჩვენებს მიმდნარე სტატუსს, არის თუ არა შეცვლილი ან ახალი ფაილები

git diff - გვაჩვენებს შეტანილ  ცვლილებებს

git fetch - დისტანციური რეპოზიტორიდან გადმოიტანს ცვლილებებს ლოკალურ რეპოზიტორიში, სამუშაო დირექტორიაში ეს არ აისახება

git pull (origin <name>) - ჯერ გააკეთებს fetch-ს და ლოკალური რეპოზიტორიდან დამერჯავს სამუშაო დირექტორიაში, სამუშაო დირექტორიაში აისახება ცვლილებებიც

git pull --rebase - ალაგებს კომიტების თანმიმდევრობას

git push (origin <name>) - დისტანციურ რეპოზიტორიზე ატვირთვა

git add remote <name> - დისტანციური რეპოზიტორის მიბმა

git chechout <branchname> - სხვა ბრენჩზე(განშტოებაზე) გადასვლა

git chechout -b <branchname> - ახალი ბრენჩის შექმნა და გადასვლა
